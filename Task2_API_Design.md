# Задание 2. Проектирование API
## GET petrushka/stores/partners
### Назначение

Метод возвращает список доступных магазинов-партнеров для отображения на экране выбора магазина в мобильном приложении интернет-магазина «Петрушка Зеленая».

### Авторизация

Не требуется.

### Логика работы

Метод получает список активных магазинов-партнеров, доступных пользователю с учетом его местоположения.
Если доступных магазинов нет, метод возвращает пустой массив.
## Входные параметры 
### Query string

| Название | Тип данных | Обязательность | Описание |
|---|---|---:|---|
| `latitude` | number | 1 | Широта местоположения пользователя или выбранного адреса доставки. Допустимый диапазон: от `-90` до `90`. |
| `longitude` | number | 1 | Долгота местоположения пользователя или выбранного адреса доставки. Допустимый диапазон: от `-180` до `180`. |
| `limit` | integer | 0..1 | Количество магазинов, возвращаемых в ответе. Значение по умолчанию — `4`. |
| `offset` | integer | 0..1 | Смещение относительно первой записи для реализации пагинации. Значение по умолчанию — `0`. |
## Выходные параметры
| Название | Тип данных | Обязательность | Описание |
|---|---|---:|---|
| `id` | uuid | 1 | Идентификатор магазина |
| `name` | string | 1 | Название магазина |
| `logoUrl` | string | 1 | URL логотипа магазина |
| `deliveryInterval` | object | 0..1 | Ближайший интервал доставки |
| `externalUrl` | string | 1 | Ссылка на внешний ресурс магазина |
| `isAvailable` | boolean | 1 | Признак доступности магазина |

## Коды ответа

| Код | Название | Описание |
|------|-------------------------|------------------------------------------------|
| 200 | OK | Список магазинов-партнеров успешно получен. |
| 400 | BAD_REQUEST | Некорректные параметры запроса. |
| 404 | NOT_FOUND | Магазины-партнеры по указанным параметрам не найдены. |
| 500 | INTERNAL_SERVER_ERROR | Внутренняя ошибка сервера. |
## Обработка перехода на внешний ресурс
При нажатии на карточку магазина мобильное приложение получает значение поля `externalUrl` соответствующего магазина и открывает ссылку во внешнем браузере.

Перед открытием ссылки приложение должно проверить:

- поле `externalUrl` заполнено;
- ссылка имеет корректный формат;
- используется протокол `https`.

Если поле `externalUrl` отсутствует или содержит некорректное значение, переход не выполняется, а пользователю отображается сообщение:

`Не удалось открыть страницу магазина`.
